Scanners

Sometimes you need to get some information from something. Maybe it's a file, or perhaps it's user input. Either way you need some code that will read this into your program. Then you can do something with it.
Enter the Scanner class! A Scanner will take in a stream of data (basically where data is progressively arriving) at the program, such as a user typing slowly, and then give the result to your code in a useful format.

In this example, we will be creating a scanner to take user input. 

        
          Scanner myScanner = new Scanner(System.In);
This creates a new Scanner called myScanner, and creates it using System.In, which is the input on the command line. This will make more sense a bit later.

So, how does this help us? Now that we have a Scanner we can pull some user input in. Enter the following code, and see what happens when you run it. 

        
          Scanner myScanner = new Scanner(System.In);
          System.out.println("Please enter your name:");
          String name = myScanner.next();
          System.out.println("Nice to meet you, " + name + "!");

Memory is typed...!

By now, you probably have realised that we have been using a keyword before declaring our variables, such as "int" or "String". These keywords simply mean what is the type of the variable that we are about to declare, meaning what type of information this variable can keep. type classifications in java are not so complicated, here are the typically used types in java

      
        int a = 10; //Integer type
        double b = 10.0; //if you have a floating point number, this is the mainly used type!
        char c = 'd'; //this is how you show characters.
        String c = "hello!"; //

remember that Java is case sensitive, so make sure you type in the given keywords in the right way. although these are only a small number of types, but for now you will be able to do any exercise we give! We will teach you more on data types and a really useful concept in java , meaning class(which is kinda realated to data types) in day 6!

What if...?

At some points you want your programme to seperate the operations based on different conditions. suppose that you want to print out the absolute value of a number. (if you have done the second part of the last day's excercises, you should be familiar with Math.abs function, if not, don't worry we'll just show it now :) ) in order to do it, you have two different conditions to consider. If your number is greater than or equal to zero, then you're good! but if not, you need to multiply your number by -1. The way of telling it to your computer is using the notations below. 

        
          Scanner myScanner = new Scanner(System.In);
          int a = myScanner.nextInt();
          if (a >= 0)
          {
            System.out.println(a);
          }
          else
          {
            System.out.println(a*-1);
          }
Also there are sometimes that you have more than two conditions, you can show it via the notations below: 
        
          if (//condition 1 goes here)
          {
            //statements for condition 1.
          }
          else if (//condition 2 goes here)
          {
            //statements for condition 2.
          }
          else
          {
            //statements for the rest of conditions
          }
You can see that you are allowed to have multiple conditions. you can also even not use the else block, and just have if blocks. (remember though at some special point you will face problem with not using the else block, we will explain it tommorow...)
