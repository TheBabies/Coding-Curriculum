Classes?!?

Previously, we have taught you how to use types in java such as int, char or double. these types are also known as primitive types. but what if you want to make your own type and try to get copies of that type just like the primitive ones we had? Here we are going to introduce a new concept named Classes a class is a blueprint, a pattern that you give in order to build new types, which from now on we call as objects. Till now you have been coding in a file named Main.java, and if you look at your codes all of them are inside a curly bracket which has the expression "public class Main" on top. Java is an object oriented language meaning every single function(or method for classes) shoud be inside a class, and you can make new classes and work with them! This might sound a little bit confusing so let's take a look at the piece of code below: 

        
          public class Dog
          {
            public String name;
            public String breed;
            public int age;

            public void bark()
            {
              System.out.println("Doge is barking!");
            }
            public void play()
            {
              System.out.println("Doge is playing!");
            }
          }

This is the class "Dog". Remember that each class should be in a file named same as your classes name, and the class name should start with a capital letter, unlike the name of variables and funcitons(we will call functions "methods" from now cause you are learning about the object oriented programming and methods are the functions that are related to a class.)
As said before, classes are like new types, but the difference between them and primitive types are that they can hold different attribures. for example, a dog in the real world has a name, a breed and also an age. a dog barks and plays. these are the features of a dog, and should be included in the dog's class. but how are we going to use these stuff?
